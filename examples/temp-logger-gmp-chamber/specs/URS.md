## 1. Introduction

This **User Requirements Specification (URS)** describes the requirements that the system **Temperature Data Logger for GMP Storage Chamber** must meet. It defines the "what" from the regulated user's perspective and constitutes the formal input to the Requirements Definition phase of the CSV V-Model.

**Scope**: continuous monitoring and recording of temperature for one (1) GMP cold-storage chamber, excursion alarm management, retention of the data as the primary GxP record, and signed periodic reports.
**Out of scope**: control/regulation of the chamber temperature (the system only monitors, it does not act on the refrigeration equipment); humidity monitoring; integration with labeling systems; external customer portal.

## 4. Project context

### 4.1 Project objective and purpose

Replace the manual chart-recorder logging of the GMP cold chamber with a validated electronic system providing continuous monitoring, tamper-evident records with audit trail, and traceable excursion handling. Validation is required and approved per [GXP-ASSESS](../../../templates/csv/GXP-ASSESS.md).

### 4.2 System description

Configured COTS product: temperature sensors in the chamber → acquisition gateway → server application (on-premises) with database → web interface for query, alarms and reports. Remote read access for QA reviewers.

### 4.3 End user

QC technicians and warehouse operators at the site (daily operation); QA reviewers (periodic review and report signing).

### 4.4 Related systems

- **Site eQMS** (receiver): receives excursion events as deviations.
- **Data historian** (receiver): receives raw temperature readings for long-term trending.

> Interfaces are described from the receiving system's perspective (canonical rule).

## 9.1 Functional requirements — `URS-FUNC`

#### 9.1.1 Detailed process description

The system acquires temperature readings at a fixed interval, stores them as the primary record, evaluates them against configured thresholds, raises alarms and notifications on excursion, and lets QA generate signed periodic reports. Operation is 24/7 and unattended except for alarm handling.

#### 9.1.2 Partial functions

| ID-No. | GxP (Y/N) | Prio. (H/M/L) | Requirement |
|---|---|---|---|
| `URS-FUNC-001` | Y | H | The system shall record the chamber temperature at a configurable interval, by default every 1 minute, with a timestamp. |
| `URS-FUNC-002` | Y | H | The system shall allow configuring temperature alarm thresholds (lower and upper limit) restricted to authorized users, with the change recorded in the audit trail. |
| `URS-FUNC-003` | Y | H | The system shall raise an alarm when a reading falls outside the configured range and notify on-call staff within ≤2 minutes of the reading. |
| `URS-FUNC-004` | Y | M | The system shall display a dashboard with the current reading and the last-24h trend for each sensor. |
| `URS-FUNC-005` | Y | H | The system shall allow a user with the QA role to generate a periodic temperature report for the selected period and sign it electronically. |
