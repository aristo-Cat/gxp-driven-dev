## 1. Introduction

This **Functional Specification** describes **how** the system **Temperature Data Logger for GMP Storage Chamber** realizes the requirements defined in its [URS](../../../templates/csv/URS.md) (specs/URS.md). It is the Functional Specification phase of the CSV V-Model and is verified in the OQ.

**Scope of this FS**: realization, by configuration of the Acme ChamberWatch COTS platform, of continuous temperature monitoring, excursion alarming/notification, tamper-evident GxP records with audit trail, electronically signed periodic reports, and validated interfaces to the site eQMS and data historian.

> As a Cat 4 system, the detailed configured settings/parameters are documented in the [Configuration Specification](../../../templates/csv/CS.md) (specs/CS.md). This FS describes the functional realization; the CS holds the concrete configuration values.

## 4. Project context

### 4.1 System Overview

Acme ChamberWatch is deployed on-premises: calibrated temperature sensors in the chamber → acquisition gateway (with local buffer) → server application + database → web UI. Realization is by **configuration** (alarm thresholds, notification rules, user roles, report templates, interface endpoints), not custom code. Two outbound interfaces: excursion events → site eQMS; raw readings → data historian. QA remote read access via the corporate IdP with MFA.
